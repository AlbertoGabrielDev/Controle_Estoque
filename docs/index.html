Por que este sistema?

Criado para resolver problemas reais de gestão de suprimentos e finanças.

Alertas Inteligentes

Itens abaixo da quantidade mínima ficam em roxo. Itens próximos do vencimento ficam amarelos, e vencidos, vermelhos.

Fluxo de Compras P2P

Requisição > Cotação > Pedido > Recebimento corporativo alimentando estoque e contas a pagar.

Financeiro Integrado

Hierarquia de Centros de Custo (pai/filho) e Contas Contábeis para categorização de despesas.

Arquitetura Limpa e Modular

O projeto foi construído usando Feature Modules em vez de uma estrutura MVC monolítica comum. Isso significa que o código relacionado a Finanças não se mistura com o código de Estoque.

  • Repository Pattern: Mocks fáceis e lógica DB abstraída.
  • Service Layer: Controllers limpos, regras no Service.
  • Inertia.js: SPA moderno sem precisar escrever uma API REST em endpoints separados.
